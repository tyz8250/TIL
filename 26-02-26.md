## 仮説（自分の理解）

```go
// 正しい書き方
contactList = make([]Contact, 0) // 空のスライスを作成,長さ0,容量0,nilではない

contactIndexByName = make(map[string]int)	// 空のマップを作成,キーはstring,値はint,nilではない
}

var contactList2 []Contact // nilとなる
var contactIndexByName2 map[string]int　// nilとなる

// nil-->panic(nil map write),読み取りはOK
contactIndexByName["John"] = 0

```

- init関数は　プログラム起動時の準備
	- 空の連絡先リストを作る
	- 名前-->indexの辞書を作る

- 以下のコードは問題ない

```go
var contactList []Contact          // nilスライス
contactList = append(contactList, x) // 問題なし

//nilスライスはappend-->OK,自動で新しい配列を作る
// panicしない
```

- mapは『ハッシュテーブル』が必要だから、nilのままでは書き込むことはできない。
- 書き込み時に内部バケット構造が必要になるため、nil状態では作業できない
	-  予期しないメモリ確保を防ぐ
	- バグを早期検出
	- 明示的初期化を強制
- sliceは『apppend時に自動で配列を作れる』設計
- `return &contactList[index]` は、そのインデックスの要素（Contact）のメモリアドレスを返している。

### 今日の本質

init関数は「プログラム開始時に必要な初期状態を保証するため」に使う。

特にmapのようにnilでは書き込みできない型の初期化に重要。
- initは初期状態を保証するための仕組み
- nilは未初期化状態であり、型ごとに挙動が違う
- mapは内部構造が必要なためmake必須
- sliceはappendで内部配列を生成できる
- &は値ではなく「場所」を返す
