① やったこと
	•	POST /requests を実装し、JSONを受け取ってリクエストを作成できるようにした
	•	curl で実際にリクエストを送り、レスポンスが返ることを確認した
	•	handler → service → repository の流れで Create が呼ばれていることを追った
	•	in-memory repository を使って「DBなし」で永続化の形を作った

  ② ハマったポイントと原因
 return 値の不一致エラー
 ```text
too many return values
have (*model.Request, error)
want (error)
```

原因
	•	service.Create が error だけ返す設計なのに
	•	repository.Create は (*model.Request, error) を返していた
	•	handler / service / repository で責務と型がズレていた
層ごとの責務をそろえる必要があると理解した

---
interface 実装エラー
```text
does not implement RequestRepository
```

原因
	•	interface と実装で
	•	引数の型
	•	戻り値（ポインタ or 値）
が一致していなかった

Goでは interface は「シグネチャ完全一致」が必要
