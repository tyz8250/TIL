## 仮説（自分の理解）
-  sliceは `[]`,`make([]int,3,5)` で作成可能
- arrayとは違い、容量を決めることができる
- appendで追加した際に、容量以上になった場合は自動的に倍の容量にするためにメモリを確保する
- `%v`はどんな型も表示する
- `%+v`は構造体やmap,sliceで便利で、フィールド名付きで表示できる
- `%d`は整数表示
- `%s`は文字列表示
- `fmt.Printf("value=%#v type=%T\n", x, x)`で大体解決するらしい
- Go にはokチェックがある
- `value, ok := star[key]`
- value --> mapから取得した値（なければゼロ値）
- ok --> キーが存在したか(true/false)
	① mapから値を取り出す
	② そのキーが存在したかを ok で確認する
	③ 存在したときだけ処理する

- nilのmap `var configs map[string]int`
	- メモリ未確保
	- 読み取りOK
	- 書き込みするとpanic
- makeしたmap
```go
configs := make(map[string]int)
configs["a"] = 1 // OK
```

## 気づき
-  fmtの%記号は暗記するものではなく、「表示ルール」として必要なものだけ使えばよい。%v / %#v / %T を中心に使えば大体対応できる
- 文法は暗記ではなく「構造」で理解すると忘れにくい
- slice / map / pointer はすべて参照を扱う概念でつながっている
- okチェックは「存在確認」の共通パターン（map / DB / API）
- nilは「未初期化状態」であり、読み取りと書き込みで挙動が違う
