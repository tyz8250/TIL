## 今日の問い（重要）
（なぜ？を1つ）
- `func divide(a, b int) (int, error)`の`(int, error)`の意味
---

## 仮説（自分の理解）
- 戻り値が2つであること（1つ目が計算結果で、2つ目がエラー情報）
- Goは例外ではなく、戻り値でエラーを返す設計

```go
return a / b, nil
```
- nil = 「エラーは存在しない」
- エラーが発生していないことを示すために、error型のゼロ値（nil）を返す

エラー時
```go
return 0, errors.New("divide by zero")
```
- 計算結果は意味がない　0
- errorには実体を入れる
- errors.New() は：　error型の値を作る関数
