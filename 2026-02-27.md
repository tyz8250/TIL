## 仮説（自分の理解）
- ベースケースを用意すること
- ベースケースがないと、stack overflow-->panicとなる
- 再帰を呼び出す処理を行うこと
- 木構造では必ず使う
- 再帰はstackが増え続けた場合、メモリをどんどん使う

```go
func sum(numbers ...int) int {
	total := 0
	for _, number := range numbers {
		total += number
	}
	return total
}
```
- `...int`は『intを0個以上受け取れる』そして関数の中では`[]int`(スライス)になる

## 思考ログ（自由）
- 以前はpythonで書いていたが、Goでも難しくない
- 可変長引数という単語を覚えた

## 気づき
- 再帰は「自己と同じ構造」に対して自然に書ける（木構造など）
- 再帰は便利だがstack消費というコストがある
- 可変長引数は内部でスライスとして扱われるため、rangeやlenが使える
- `...` は「可変長受け取り」と「スライス展開」の2つの意味を持つ
