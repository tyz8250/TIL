## 仮説（自分の理解）
- `fmt.Sprintif` は文字列を作る関数
- 以下で比較してみる
```go
fmt.Println("hello") //画面に表示する

s := fmt.Sprintf("hello") //文字列を作って変数に入れる
```

 例で考える
``` go
name := "Alice"
fmt.Sprintf("Hello %s", name) // 文字列を作る
fmt.Println(s)　//　出力する

// Hello Alice
```
もう一つ例
```go
e.InputA = 10
fmt.Sprintf("a=%d", e.InputA)
// %d はフォーマット指定子。 int型を指定
// "a=10"
```

- 以下の問題を考える
```go
func (e MathError) Error() string
// ↑と書くだけで、以下が成立するかどうか
var err error = MathError{...}
```

- GoのError型の決まりについて
```go
// 以下の決まりがある。
type error interface {
    Error() string
}
// Error()stringを持つ型はerrorとして扱えるので、以下は成立
var err error = MathError{...}
```
- Goは『明示的implements』は不要
- Goは形が合えば、インターフェイスを満たす
	-->これを構造的型システムという。

## 気づき
- errorはinterface型である
- Error() string を持つ型は自動的にerrorになる
- Goは『名義的型付け』ではなく『構造的型付け』
- fmt.Sprintfは「出力」ではなく「文字列生成」
