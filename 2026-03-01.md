## 仮説（自分の理解）
- deferは『関数の終了直前に実行される処理を予約する仕組み』
```go
func example(){
	defer fmt.PrintLn("world")
	fmt.Println("hello")
}

// 出力　
// hello
// world
```
- deferを連続した場合、LIFOになり、後入れ先だしの処理となる。
- deferは宣言時の引数が評価される
```go
x := 10
defer fmt.Println(x)
x = 20

// 出力　
//→ deferは「呼び出しを予約」するが
//→ 引数はその場で評価される
```
- deferは　正常終了,return,panic どの場合でも実行される
- panic()はGoの組み込み関数
```go
panic("error") //　異常は起きた。即座に処理を止める 
```

- recoverもGoの組み込み関数であり、『deferの中でしか意味を持たない』
```go
func recoverable() {
	defer func() {
		if r := recover(); r != nil {
			fmt.Println("Recovered from panic:", r)
		}
	}()
}
// 1.関数終了時にこの無名関数を実行
// 2.もしPanicが発生していたら
// 3.recover()で捕まえる
// 4.プログラムを止めずに続行
```
- recoverの戻り値
```go
r := recover()
// panicがあれば-->その値
// panicがなければ-->nil
```

## 気づき
- deferは関数終了直前に実行される
- 複数あるとLIFOになる
- 引数はdefer宣言時に評価される
- 主な用途はリソース解放
- panicが発生してもdeferは実行される
- recoverはdeferの中でのみ有効
- panicは想定外エラー、errorは想定内エラー

### 今日の暫定答え

deferは「関数終了時の後処理を保証する仕組み」。
リソース解放やpanic時の安全確保のために使われる。
