---

## 仮説（自分の理解）
```go
type Employee struct {
	ID        int
	FirstName string
	LastName  string
	Position  string
	Salary    int
	IsActive  bool
	JoinedAt  time.Time
}
// ポインタレシーバー(e *Employee)を使用した場合、元の値を指す
// Employee の場所を受け取る(コピーしない、元の構造体を変更dきる)
func (e *Employee) FullName() string {
	return e.FirstName + " " + e.LastName
}

// 値レシーバ(e Employee)を使用した場合、コピーが渡る
// Employee が丸ごとコピーされる
func (e Employee) FullName() string {
	return e.FirstName + " " + e.LastName

}
// 正直変更しないなら、値レシーバでもOK
```
- なぜポインタを使うか
- コピーコスト（メモリを使う、パフォーマンス落ちる）
- 一貫性（変更するメソッドがあるなら、全部ポインタに統一することが多い）
- インターフェイスとの関係性
```go
func (e *Employee) Error() string
//  *Employee は error になる
//  Employee はならない
```

- フィールド変更するならポインタ
- 大きい構造体はポインタが効率的
