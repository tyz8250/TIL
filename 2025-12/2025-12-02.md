Go Udemy

インターフェイスは、“このメソッドを持つものだけ受け取るよ”という型チェックツール。
構造体がそのメソッドを持っていれば、勝手にインターフェイスとして扱える。

```go
type bot interface {
	getGreeting() string
}
```

bot インターフェイスは"getGreeting()を持つもの"と定義。
どんな型でもこのメソッドを持っていれば bot として扱われる。

englishBot と spanishBot は構造体

```go
type englishBot struct{}
type spanishBot struct{}
```

それぞれが getGreeting を「実装している」ので bot として扱える

```go
func (englishBot) getGreeting() string {
	return "Hi There!"
}
func (spanishBot) getGreeting() string {
return "Hola!"
}
```

printGreeting は bot インターフェイスだけ見てる

```go
func printGreeting(b bot){
	fmt.Println(b.getGreeting())
}
```

💡 今日の理解まとめ
• interface = 型に実装を強制する“ルール”だけ
• 構造体がそのメソッドを持っていれば自動でインターフェイスを満たす
• 関数は「インターフェイス型」を受け取ることで
→ 共通メソッドを持つ“どんな型でも受け取れる” ようになる
• Go は“暗黙的”実装のため、実装宣言は不要
